# WhatsApp 网页版翻译

[English](README.md) | [中文](README_zh.md)

一个低侵入的 Chrome 扩展,把你在 [WhatsApp 网页版](https://web.whatsapp.com) 上收到的消息
翻译成你自己的语言,由你自己控制的大模型驱动。自带 API Key。除了你配置的那个大模型服务商,
不向任何地方上传数据。

- **实时或点击翻译。** 自动翻译每一条收到的消息;也可关掉自动、只点某一条来翻。会话内有个小控件
  按会话切换。
- **只翻对方的。** 你自己发的消息不会被翻译。可选:把你正在写的草稿翻成对方的语言再发。
- **带上下文。** 引用回复会把被引用的消息作为上下文传给模型,指代能对上。还能选**语气**
  (自然/友好/专业/活泼/简洁)。
- **低侵入。** 译文以一行小字显示在原气泡下方,适配浅色/深色主题,不改写、不遮挡原文。
- **有缓存、不重复计费。** 译文会缓存(会话内内存 + 持久化),滚动、重进会话、刷新页面都不会
  对同一段文本重复调接口。
- **可选本地知识库。** 把每条消息存进本地、可导出,喂给你自己的工具。不导出就不出机器。

服务商:**DeepSeek**(默认)、任意 **OpenAI 兼容**接口、或 **Anthropic (Claude)**。

---

## 工作原理

内容脚本监听当前会话。有新消息到达(手动模式下则是你点击某条时),把文本连同一个针对聊天调过的
提示词发给你配置的大模型,再把结果作为一行小字显示在原消息下方。真正的网络请求在扩展的后台
service worker 里发,所以你的 API Key 不接触页面。

WhatsApp 网页版会混淆并频繁改动 CSS 类名,所以扩展只认**稳定的语义信号**:

- 一条消息 = 带 `data-testid="conv-msg-<id>"` 的容器;其不透明的 `data-id` 作为去重/缓存键。
- `div.copyable-text[data-pre-plain-text]` 是文本网关——有它才是真正的文字(排除媒体/语音/贴纸/系统行)。
- 方向:**你自己发**的消息 `msg-meta` 里带已读/送达状态图标,**对方**的没有。用这个区分「你/对方」,
  不靠会变的类名。
- 文本用**保留 emoji** 的遍历读取(`img.alt`),排除引用回复的预览。

若 WhatsApp 改了结构、什么都匹配不上,控制台会打印 `[WAT] selector self-test: …`,让「坏了」被看见
而不是悄悄失效。

## 安装

1. 下载本仓库(clone,或下载 ZIP 解压)。
2. 打开 Chrome,进入 `chrome://extensions/`。
3. 打开右上角 **开发者模式**。
4. 点 **加载已解压的扩展程序**,选本扩展文件夹。
5. 打开扩展 **选项**(右键图标,或在 `chrome://extensions/` 里),配置服务商和 API Key。

> **改了扩展代码后**,要在 `chrome://extensions/` 点扩展卡片的**重载(↻)**按钮,再刷新 WhatsApp
> 标签页。**只改了选项设置**,刷新标签页即可。

## 配置

打开选项页。

### 服务商

| 服务商 | 需要填 | 说明 |
|--------|--------|------|
| **DeepSeek**(默认) | 模型、API Key | 用 `https://api.deepseek.com/v1` + `deepseek-chat`,快、便宜、适合聊天。填 Key 即可。 |
| **OpenAI 兼容** | Base URL、模型、API Key | 支持 OpenAI、Groq、Azure,或任意本地/自建的 OpenAI 兼容接口。填自定义 Base URL 时保存会申请一次该域名权限。 |
| **Anthropic (Claude)** | 模型、API Key | `claude-haiku-4-5` 最快最便宜、适合做默认;追求质量用 `claude-sonnet-5` 或 `claude-opus-4-8`。直连,带浏览器访问头。 |

API Key 存本地(`chrome.storage.sync`),只发给你选的服务商。DeepSeek Key 在
[platform.deepseek.com](https://platform.deepseek.com) 拿。

### 语气(Style)

| 语气 | 效果 |
|------|------|
| 自然(默认) | 像跟朋友发消息 |
| 友好 | 温暖、亲和 |
| 专业 | 礼貌、仍自然 |
| 活泼 | 俏皮、带点幽默 |
| 简洁 | 短、直接 |

收发都生效。模型按语境调措辞,不逐字直译。

### 阅读(对方的消息)

| 设置 | 说明 | 默认 |
|------|------|------|
| 翻译成 | 你阅读用的语言(列了 16 种;源语言模型自动识别) | 中文（简体） |
| 何时翻译 | `自动`(每条收到的)或 `点击`(只翻你点的) | 自动 |
| 每日上限 | 限制每天自动翻译次数,防止群聊烧钱(`0`=不限;点击翻译不受限) | 0 |

### 写作(你的草稿)——可选,默认关

| 设置 | 说明 | 默认 |
|------|------|------|
| 启用 | 显示浮动翻译按钮,并启用 `Alt+T` 快捷键 | 关 |
| 草稿翻译成 | 对方的语言 | 英语 |

### 知识库——可选,默认关

把每条文本消息(收+发)存进可导出的本地库。见下面 [知识库](#知识库)。

## 使用

1. 打开 [web.whatsapp.com](https://web.whatsapp.com) 正常登录。
2. 打开一个会话。

**阅读。** 自动模式下,收到的消息随到随译,每条下方出现一行译文。会话顶部有个小控件
(`Translate: Auto → …` / `Translate: Off`)按会话开关自动翻译。关掉后,点任意一条对方消息
(或它的小 `Translate` 钮)只翻这一条,再点隐藏/显示。

**写作(启用后)。** 用你自己的语言打好草稿,光标在输入框时按 **`Alt+T`**,或点浮动的
**`译 → …`** 按钮。草稿会被**替换**成译文,你检查后发送。想找回原文按 `Cmd/Ctrl+Z`。发送始终**手动**——
扩展绝不替你发。

## 知识库

在 **选项 → Knowledge base → Capture** 打开。之后你收发的每条文本消息都会存进本地库,按消息 id
去重,有译文时一并挂上。

- **Export JSON** 下载 `whatsapp-kb.json`。
- **Clear** 清空本地库。
- **Saved** 显示当前条数。

每条记录:

```json
{
  "id": "3AB0F95BC932F1FE78DF",
  "chat": "Alice",
  "direction": "in",              // in=对方  out=你
  "sender": "Alice",
  "text": "What's name",
  "quoted": "I also have one.",    // 被引用的消息(引用回复才有)
  "translation": "你叫什么名字?",
  "time": "22:57, 2026年7月14日",
  "capturedAt": "2026-07-15T14:05:22.000Z"
}
```

全程**只在本地**——扩展不上传。导出 JSON 后用你自己的凭证推进你自己的工具(Obsidian、Notion、
飞书多维表格…),这样第三方 token 不进扩展。聊天可能敏感,导出文件请保管好。

## 隐私

- 扩展没有后端。消息文本只用你的 Key 发给你配置的服务商(DeepSeek / OpenAI 兼容 / Anthropic)。
- API Key 和设置在 `chrome.storage.sync`;译文和知识库在本机 `chrome.storage.local`。
- 只有你(或自动模式)选择翻译的消息才会被发送。非文本、你自己的消息(除非用草稿翻译)、系统提示都忽略。
- 由于文本会离开本机发给服务商,敏感会话请谨慎开自动翻译,除非你接受该服务商的数据处理方式。

扩展只读取并注入 `web.whatsapp.com`——不碰 WhatsApp 协议、不替你发消息、不做任何自动化,纯客户端运行。

## 缓存行为

| 场景 | 行为 |
|------|------|
| 会话内滚动/重绘 | 从内存缓存(按消息 id)贴回——**不调接口** |
| 切走会话再回来(不刷新) | 内存缓存——**不调接口** |
| 刷新标签页 | 持久缓存(按 文本+语气+语言+模型 哈希,最多 10 万条)——翻过的相同文本**不重调** |
| 转发的相同内容、跨会话 | 持久缓存命中——**不调接口** |
| 换语气/语言/模型 | 缓存键变了,按新设置重翻一次 |

持久缓存在内存里维护,短防抖后写入 `chrome.storage.local`,重度聊天也不会频繁抖动存储。

---

## 局限

- WhatsApp 网页版会不定期改结构。若某次更新后译文不再出现,可能要更新选择器——控制台的 `[WAT]`
  自检警告就是信号。
- 只翻文本。图片、贴纸、语音、文件、已删除的消息都跳过。
- 质量和速度取决于你选的模型。

## 开发

内容脚本的整条流水线(注入、方向判断、跳过规则、emoji、引用上下文、虚拟化回收、缓存自愈、点击模式、
草稿替换、知识库采集)在真实的无头 Chrome 里对着一份模拟 WhatsApp DOM 做了验证,无需登录 WhatsApp:

```bash
npm install        # 安装 playwright-core(用你系统里的 Chrome)
npm test           # 运行 test/harness.mjs
```

`test/fixture.html` 还原了真实 WhatsApp DOM(混淆类名、`conv-msg` 容器、`msg-meta` 状态图标判方向、
引用回复);`test/harness.mjs` 驱动它并断言整条流水线。

